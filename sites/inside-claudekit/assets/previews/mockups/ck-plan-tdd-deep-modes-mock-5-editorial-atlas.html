01 / premise

Đây không phải task đổi syntax

Rust sang Go không chỉ là rewrite. Lifecycle, streaming, cancellation, process cleanup, SQLite wrapper và runtime concurrency đều đổi. Sai một điểm thì nhẹ là không build, nặng hơn là race condition chỉ lộ sau khi chạy thật.

Plan sai ở phase sớm kéo theo sửa sai ở cả implementation.
02 / distinction

Mode và flag phải tách bạch

--deep là modeĐổi pipeline, thêm researcher, per-phase scout, red-team review và validation.
--tdd là flagKhông đổi mode selection, nhưng thêm Tests Before, Refactor, Tests After, Regression Gate.
03 / deep mode

Deep cho bản đồ phase

--deep hợp khi scope là major refactor, nhiều vùng code, hoặc architectural debt. Nó ép mỗi phase có file inventory, test scenario matrix, dependency map, function/interface checklist.

Research2-3 researcher agents cho high-level architecture analysis.
ScoutPer-phase scout xác định file, test coverage, dependency edge.
OutputPhase file cụ thể hơn, cook ít phải đoán scope.
04 / tdd flag

TDD cho behavior đang chạy

--tdd đáng bật khi refactor hoặc port có behavior cũ cần giữ. Với greenfield, nó dễ thành nghi thức rỗng vì chưa có hành vi hiện tại để capture.

Tests Before -> Refactor -> Tests After
Regression Gate: go test ./... && go vet ./...
05 / choose

Quy tắc chọn nhanh

Thiếu context/ck:scout trước.
Approach mơ hồ/ck:brainstorm trước.
Major refactor/ck:plan --deep.
Behavior cần giữAny mode + --tdd.
06 / pitfalls

Các bẫy dễ gặp

P1. Plan có --tdd, cook quên flag.

P2. Dùng --deep cho task nhỏ.

P3. Dùng --tdd cho greenfield.

P4. Tin phase file tuyệt đối, không kiểm dependency.

P5. Capture bug cũ thành expected behavior.

P6. Kết hợp parallel khi ownership chưa rõ.
