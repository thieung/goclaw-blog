01 / classify

Mode khác flag ở đâu

--deep là mode: đổi pipeline, thêm researcher và scout theo từng phase. --tdd là flag: không đổi mode selection, nhưng inject cấu trúc tests-first vào phase file và cook flow.

Mode--fast, --hard, --deep, --parallel, --two. Chọn một mode chính cho pipeline.
Flag--tdd, --no-tasks. Compose với mode để thêm ràng buộc hoặc bỏ task hydration.
02 / map

--deep cho đại tu

Với port đa-stack, lỗi không nằm ở syntax mà ở hành vi runtime: process cleanup, stream buffering, SQLite migration, event emission, và lifecycle của subprocess.

Architecture2-3 researcher agents đọc docs, source, stack mới.
Per phaseInventory file create/modify/delete, test coverage, function checklist.
ReviewRed-team và post-plan validation trước khi cook.
03 / guard

--tdd khóa behavior

Flag này hữu ích khi code cũ đang chạy và refactor dễ drift. Tests Before ghi lại behavior hiện tại, Refactor đổi implementation, Tests After kiểm tra lại cùng regression gate.

Step 3.T: Write tests for CURRENT behavior
Step 3.I: Implement refactor
Step 3.V: Run regression gate

05 / pipeline

Chi phí đổi lấy ít bất ngờ

--deep đắt vì thêm researcher và per-phase scout. --tdd nhẹ hơn, chủ yếu đổi template và thứ tự cook. Hai thứ không overlap nên dùng chung được.

06 / avoid

Pitfalls

P1
Plan có --tdd nhưng cook quên flag.
P2
--deep cho task nhỏ dưới 5 file.
P3
--tdd cho greenfield không có behavior cũ.
P4
Tin phase file 100%, không tự kiểm dependency.
P5
Capture luôn bug cũ thành regression.
P6
Dùng parallel khi ownership và test isolation chưa rõ.
