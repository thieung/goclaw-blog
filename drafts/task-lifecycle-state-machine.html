Flags quyết định hành động post-turn

Trước v2.8.0, hệ thống gọi auto-complete ngay khi member agent kết thúc turn — kể cả khi agent chỉ mới bình luận hoặc escalate. v2.8.0 giới thiệu TaskActionFlags: struct theo dõi từng loại hành động trong một turn, sau đó switch/case để ra quyết định chính xác.

Go internal/tools/team_task_action_flags.go 
// TaskActionFlags tracks which team_tasks actions a member agent took during a turn.
// The consumer reads these flags post-turn to decide whether to auto-complete,
// skip, or re-dispatch the task.
type TaskActionFlags struct {
  Completed  bool // team_tasks(action="complete")
  Reviewed   bool // team_tasks(action="review")
  Escalated  bool // team_tasks(action="comment", type="blocker")
  Progressed bool // team_tasks(action="progress")
  Commented  bool // team_tasks(action="comment")
  Claimed    bool // team_tasks(action="claim")
}

Hồi phục tự động

Task bị kẹt tự giải quyết

v2.8.0 giới thiệu hai cơ chế hồi phục nền: stale detection cho task in_review quá lâu, và orphaned recovery cho task blocked mà tất cả blocker đã kết thúc.

Stale Detection
Task ở trạng thái in_review quá 4 giờ bị đánh dấu stale. TaskTicker chạy nền, quét định kỳ và phát event team.task.stale để leader xử lý.
Orphaned Recovery
Task blocked mà tất cả blocker đã vào trạng thái terminal (completed/cancelled) được tự động tái dispatch — bảo toàn chuỗi blocking phụ thuộc.
