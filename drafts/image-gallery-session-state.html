01 — Image Gallery

Browse all images with keyboard navigation

Before v2.21.0, clicking an image in chat opened a single-image lightbox. To see another image you had to close and click again. Conversations with screenshots, diagrams, or AI-generated images were painful to review.

Now every image in a conversation is registered in a ChatImageGalleryProvider context. Click any image to open the lightbox — then navigate the entire conversation's images with arrow keys or the prev/next buttons.

// Wraps chat thread — provides conversation-wide image lightbox
export function ChatImageGalleryProvider({ images, children }) {
  const [currentIndex, setCurrentIndex] = useState<number | null>(null);

  const openImage = useCallback((src) => {
    // Normalize: strip query params for matching
    const clean = (s) => s.split("?")[0] ?? s;
    const idx = images.findIndex((img) => clean(img.src) === clean(src));
    setCurrentIndex(idx >= 0 ? idx : null);
  }, [images]);

  return (
    <Ctx.Provider value={{ openImage, allImages: images }}>
      {children}
      {currentIndex !== null && (
        <ImageLightbox
          images={images}
          currentIndex={currentIndex}
          onNavigate={setCurrentIndex}  // arrow key nav
        />
      )}
    </Ctx.Provider>
  );
}
02 — Session State Restore

Running indicators survive session switches

In v2.20 and earlier, switching from a running session to another and then back would lose the IsRunning indicator. The UI showed the session as idle even when an agent was still actively working.

v2.21.0 introduces a session.status RPC that the client calls on every session switch. The server returns the current IsRunning and activity state, which the frontend restores immediately — no polling delay, no visual glitch.
